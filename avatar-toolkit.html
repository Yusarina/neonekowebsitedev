Avatar Toolkit

What is Avatar Toolkit?

Avatar Toolkit will be a more modern toolkit for your avatar needs. The idea is that Avatar Toolkit will replace the Cats Blender Plugin from Blender 5.0+. We want to include most features of Cats and more without the need for most 3rd-party add-ons such as Material Combiner and MMD Tools.

You can follow the development here: GitHub Repository

Why are we making Avatar Toolkit?

After taking over Cats several months ago, we started to notice that relying on 3rd-party add-ons was getting harder and harder. Some add-ons are no longer updated or are very slow at getting updates. This was affecting our updates to Cats, and we were having to put warnings on the Blender 4.x version of Cats as some 3rd-party add-ons were not updated.

This led to us also hotfixing stuff on these 3rd-party add-ons and more. Therefore, we decided it would be better for us to integrate most 3rd-party functions into a new plugin, fix the code, and make it better as well. That's where Avatar Toolkit was born.

Also, we think it's best to move away from a name which is associated with the original author of Cats, who still to this day takes money from their Patreon for an abandoned plugin which they haven't worked on in 3 years.

Can I use Avatar Toolkit?

Although the Toolkit loads and works on Blender 4.3 Alpha, we highly recommend not using it as it's in a VERY early stage of development. A lot of features are missing and there are issues even with the current added features. We are aiming for a stable release for Blender 5.0. If we need some people to test, we will announce it.

Will Avatar Toolkit use MMD Tools?

No, our aim is to completely avoid MMD Tools. The issue with MMD Tools is that the author only cares about making their add-on work with full LTS releases of Blender and doesn't support non-LTS releases of Blender.

An example of an LTS release is Blender 4.2, which is an upcoming LTS release of Blender so the author has been pretty on the ball with releasing updates for it. However, 4.0, 4.1, and 4.3 are not LTS releases of Blender. Therefore, the author of MMD Tools has no interest in making the plugin fully work with those versions of Blender.

This is an issue because we do prefer to support all new Blender releases for two reasons:

  1. It makes it easier to update if we just do it for each release.
  2. Most people are using the Steam version of Blender, which automatically updates to the newest version. Therefore, we get A LOT of requests for the plugin to be updated.

If MMD Tools isn't updated to a non-LTS version of Blender, then any function we rely on for MMD Tools could just break, which means we have to fix MMD Tools ourselves, which isn't ideal. Cats also wasn't even using 90% of MMD Tools functions, so it's just extra bloat on top of that.

Therefore, we are going our own custom route and making our own ways to import MMD models and to just create our own MMD functions instead.
